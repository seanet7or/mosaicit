5. Troubleshooting

5.1 Rendering the Mosaic Fails

Error message Workaround
Error - no matching tile was found! [...] Choosing a image database with more entries might also remove the problem. You have set a minimum distance between two occurances of the same tile in the mosaic or you have set a value for the maximum number of occurances for the tile in the mosaic. Try decreasing the minimum distance or increasing the maximum number of occurances or uncheck the options. You could also use a database with more entries.
Error loading the tile image "FILENAME"! Propably the image was deleted or changed into an incompatible format after adding it to the selected database. If this is the case, loading the database in the "Edit database" dialog and updating it there will resolve the problem.
It might also be that the file is on a removable storage device that is not plugged in. If so, plug it in and restart the process.
If this doesn't help, perhaps your computer ran out of memory. Try creating a photographic mosaic with less megapixels then.
Error reserving memory for the mosaic image! Your computer ran out of memory. Try creating a photographic mosaic with less megapixels.
Error reserving memory for the tiles map! Your computer ran out of memory. Try creating a photographic mosaic with less megapixels.
Error scaling the original picture! Propably your computer ran out of memory. Try creating a photo graphicmosaic with less megapixels.
Error scaling the tile image "FILENAME"! Propably your computer ran out of memory. Try creating a photo graphicmosaic with less megapixels.
Error loading the original image! Check if the original image you selected exists and if it is a valid image file in a supported format (jpg, bmp, tiff).
